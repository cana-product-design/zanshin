# Design Tokens

All Zanshin components use CSS custom properties exclusively. No hardcoded hex values appear in component code.

## Spacing Scale — Ma (間)

The spacing scale encodes the Japanese concept of *ma* — the meaningful space between things.

| Token | Value | Use |
|---|---|---|
| `--space-1` | `4px` | Micro gaps |
| `--space-2` | `8px` | Icon padding, badge gap |
| `--space-3` | `12px` | Button padding (sm) |
| `--space-4` | `16px` | Input padding, button padding (md) |
| `--space-6` | `24px` | Component internal spacing |
| `--space-8` | `32px` | Card padding |
| `--space-sm` | `16px` | Component-to-component |
| `--space-md` | `24px` | Section inner padding |
| `--space-lg` | `40px` | Between content blocks |
| `--space-xl` | `64px` | Between major sections |
| `--space-2xl` | `96px` | Page section separation |

## Border Radius

**Hard ceiling: 8px.** Soft forms contradict budo geometry.

| Token | Value | Use |
|---|---|---|
| `--radius-sm` | `2px` | Inputs, badges, tags |
| `--radius-md` | `4px` | Buttons, dropdowns |
| `--radius-lg` | `8px` | Cards, modals, panels |

## Motion — Zanshin Transitions

Every interaction leaves a trace of awareness.

| Token | Value | Character |
|---|---|---|
| `--ease-zen` | `cubic-bezier(0.4, 0.0, 0.2, 1)` | Deliberate, revealing |
| `--ease-kendo` | `cubic-bezier(0.25, 0.46, 0.45, 0.94)` | Controlled, micro-interactions |
| `--ease-strike` | `cubic-bezier(0.0, 0.0, 0.2, 1)` | Fast in, soft out — decisive |
| `--duration-short` | `300ms` | Minimum — no transition under this |
| `--duration-slow` | `400ms` | Standard component transitions |
| `--duration-medium` | `500ms` | Reveals, page elements |
| `--duration-long` | `800ms` | Scroll reveals, yugen moments |
