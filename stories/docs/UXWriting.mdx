# UX Writing — 言霊 Kotodama

**Every word is a threshold.**
The *torii* gate marks the boundary between profane and sacred — it does not explain it.
Copy works the same way: it marks what the interface already communicates and steps aside.

---

## Three Pillars

### 1. Smart Brevity
Readers decide in the first sentence whether to keep reading. State the finding first. Everything else is context for those who want it. One idea per unit. Active voice. Cut anything that could be deleted without losing meaning.

### 2. Shinto — 神道 Threshold Principle
Copy confirms what the design communicates. It does not narrate it. A button is a gate (*torii*) — it marks passage, it does not describe the journey. Silence is valid. Names carry power — *kotodama* (言霊, word spirit) means naming is ritual. Once a label is established, changing it without reason breaks the ceremony.

### 3. Buddhism — 仏教 Impermanence
Errors are temporary states, not catastrophes. Suffering arises from attachment to outcomes that were never fixed. The middle path in tone: not clinical, not effusive. Present. Grounded. Clear. The exit completes the form — *zanshin* (残心) means lingering awareness: a toast, a confirmation, a final state are not afterthoughts. They close the cycle.

## Voice at a Glance

| Moment | Zanshin voice | Not this |
|---|---|---|
| CTA | Begin | Click here to get started |
| Error | Build failed — 3 TypeScript errors. | Oops! Something went wrong. |
| Success | Saved. | Awesome! Your changes are saved! |
| Loading | Building Storybook... | Please wait while we load... |
| Empty state | No components. Add the first one. | There's nothing here yet! |
| Tooltip | Two modes, one root. | This feature allows you to... |
| APCA violation | Asagi on ao-sumi — prohibited. Lc −39.6. | Low contrast detected. |

## Forbidden Patterns

These violate all three pillars simultaneously:

| Avoid | Because |
|---|---|
| *Oops!* | Infantilizes the error and the user |
| *Don't worry...* | Creates the worry it tries to prevent |
| *Simply* / *Just* | Condescending when the thing is not simple |
| *Please wait...* | Ceremony around a system state |
| *We're working hard to...* | The user wants the result, not the effort narrative |
| *Awesome!* / *Amazing!* | Enthusiasm as noise |
| *Click here* | Non-descriptive — violates *kotodama* |
| Passive error constructions | *"An error was encountered"* — own the state |
