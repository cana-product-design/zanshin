# Accessibility — APCA Audit

Zanshin uses the **Advanced Perceptual Contrast Algorithm (APCA)**, the contrast model proposed for WCAG 3.0. Unlike WCAG 2.x's single ratio, APCA produces a signed **Lc** value that distinguishes text-on-dark (WoB) from text-on-light (BoW).

The APCA test suite runs on every CI push and must pass all 29 tests before merge.

---

## Lc Thresholds

| Lc (absolute) | Use case |
|---|---|
| ≥ 75 | Body text at any size |
| ≥ 60 | Large text (24px+), Fluent UI labels |
| **≥ 45** | **Interactive elements — buttons, links, inputs, nav** |
| ≥ 30 | Decorative / incidental text |
| ≥ 15 | Non-text elements (borders, icons, dividers) |
| < 15 | Fail — insufficient for any purpose |

## Key Design Rules from the Audit

1. **Never use asagi as text on light surfaces.** Lc +43.1 fails the interactive threshold. Use aiiro (Lc +77.0) for links on light backgrounds.

2. **Never use aiiro text on kachiiro backgrounds.** Lc −15.1 — same hue family, no perceptual contrast. Use asagi (Lc −48.9) instead.

3. **Aiiro buttons require white text.** White at Lc −78.5 is safe. Shironeri at Lc −72.0 is large-text only. Dark text on aiiro buttons fails.

4. **Gold accent is decorative on light surfaces.** Kintsugi gold (Lc +40.4) does not meet the interactive threshold. Never use as body link text.

---

## Minimum Size Requirements

When a pairing falls in the 45–74 Lc range, font size constrains safe use:

| Lc (abs) | Normal weight minimum | Bold/semibold minimum |
|---|---|---|
| 75+ | 12px | 10px |
| 60–74 | 18px | 14px |
| 45–59 | 24px | 16px |
| 30–44 | Non-text only | 24px (with caution) |

## Running the APCA Test Suite

```bash
npm run test:apca
```

29 tests covering:
- Lc annotation accuracy (±1.0 tolerance)
- Passing pairs clear their declared threshold
- Intentional failures stay below their threshold
- APCA engine self-tests (black/white reference values)

All 29 tests must pass. The CI workflow (`apca-ci.yml`) enforces this on every push.
